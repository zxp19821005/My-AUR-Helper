# Plan: 新增 commands/files.rs 和 commands/sys_command.rs

## Context

当前后端 commands 层缺少文件操作系统命令执行两个模块。需要新增两个模块：
1. **`files.rs`** — 通用文件操作（拷贝、删除、移动、目录扫描、PKG 文件名解析）
2. **`sys_command.rs`** — 通过 tauri-plugin-shell 执行系统命令（pacman、makepkg）

## 设计

### 1. `commands/files.rs` — 文件操作

使用 `tokio::fs` 异步文件 I/O，PKG 文件名解析用正则。

```rust
// ── Tauri 命令 ──

copy_file(src: String, dst: String) -> Result<(), String>
move_file(src: String, dst: String) -> Result<(), String>
delete_file(path: String) -> Result<(), String>
delete_directory(path: String) -> Result<(), String>
create_directory(path: String) -> Result<(), String>
read_file(path: String) -> Result<String, String>
list_directory(path: String) -> Result<Vec<DirEntry>, String>
file_exists(path: String) -> Result<bool, String>
file_metadata(path: String) -> Result<FileMetadata, String>
scan_pkg_files(directory: String) -> Result<Vec<PkgFileInfo>, String>
batch_delete(paths: Vec<String>) -> Result<BatchDeleteResult, String>

// ── 辅助结构体 ──

struct DirEntry { name, is_dir, size, modified_at }
struct FileMetadata { size, modified_at, is_dir, permissions }
struct PkgFileInfo { filename, epoch, pkgrel, arch, size }
struct BatchDeleteResult { deleted: usize, errors: Vec<String> }

// ── PKG 文件名解析逻辑 ──
// 文件名格式: {name}-{version}-{pkgrel}-{arch}.pkg.tar.zst
// 示例: gitify-bin-1.0.0-1-x86_64.pkg.tar.zst
// epoch 通常为 0，特殊格式: 1:1.0.0-1-x86_64.pkg.tar.zst
fn parse_pkg_filename(filename: &str) -> Option<PkgFileInfo>
```

### 2. `commands/sys_command.rs` — 系统命令（通过 tauri-plugin-shell）

使用 `tauri_plugin_shell::ShellExt` 或 `app.shell().sidecar()` 执行命令。

```rust
// ── Tauri 命令 ──

run_command(command: String, args: Vec<String>) -> Result<CommandOutput, String>
install_package(pkgname: String) -> Result<CommandOutput, String>     // sudo pacman -S --noconfirm
remove_package(pkgname: String) -> Result<CommandOutput, String>     // sudo pacman -R --noconfirm
clean_cache(keep_versions: usize) -> Result<CommandOutput, String>   // sudo pacman -Sc --noconfirm
get_package_version(pkgname: String) -> Result<String, String>       // pacman -Qi Version
list_installed_packages() -> Result<Vec<String>, String>             // pacman -Qq
sync_database() -> Result<CommandOutput, String>                     // sudo pacman -Sy
makepkg(directory: String, args: Vec<String>) -> Result<CommandOutput, String> // makepkg in dir

// ── 辅助结构体 ──

struct CommandOutput { exit_code: i32, stdout: String, stderr: String }
```
