# Frontend UI Modernization Plan

## Overview
Redesign the My-AUR-Helper frontend with a modern layout: collapsible sidebar, VS Code-style tab bar, per-page toolbar, and settings in a separate Tauri window.

## Implementation Steps

### Step 1: Install Dependencies
```bash
pnpm add lucide-vue-next
```

### Step 2: Add Lucide Icons to Sidebar

Replace emoji icons with Lucide icons in the sidebar nav items.

### Step 3: Create Tab Store (`src/stores/tabs.ts`)
- `openTabs`: array of `{ path, label, icon }`
- `activeTab`: current path
- `openTab(path)`: add tab if not exists, set active
- `closeTab(path)`: remove tab, switch to adjacent
- `switchTab(path)`: set active

### Step 4: Create Sidebar Component (`src/components/Sidebar.vue`)
- Props: `collapsed: boolean`
- Emits: `toggle`
- When collapsed: show only icons (48px wide)
- When expanded: show icons + labels (200px wide)
- Toggle button at top/bottom
- Use Lucide icons for each nav item
- Hover tooltip when collapsed

### Step 5: Create TabBar Component (`src/components/TabBar.vue`)
- Read tabs from `tabs` store
- Each tab shows icon + label + close button
- Click tab вҶ’ switch active
- Click close вҶ’ remove tab
- Click "+" or navigate вҶ’ open new tab
- Last tab cannot be closed

### Step 6: Create PageToolbar Component (`src/components/PageToolbar.vue`)
- Props: `title: string`
- Slot for page-specific actions (left side)
- Settings gear button on right вҶ’ opens Tauri window

### Step 7: Refactor App.vue
- Remove old sidebar, replace with `<Sidebar>` + `<TabBar>` + `<PageToolbar>` layout
- Watch route changes to auto-open tabs
- Wire up settings button to open new window

### Step 8: Update Each View
Remove the `<div class="page-header">` from each view since the toolbar is now in the layout. Each view only keeps its content.

Files to modify:
- `src/views/Dashboard.vue`
- `src/views/PackageList.vue`
- `src/views/PackageDetail.vue`
- `src/views/BackupManager.vue`
- `src/views/ProxySettings.vue`
- `src/views/Settings.vue`
- `src/views/LicenseManager.vue`
- `src/views/LanguageManager.vue`
- `src/views/LogViewer.vue`

### Step 9: Settings Window via Tauri
- Add `webview:allow-create-webview` and `webview:allow-create-webview-window` to capabilities
- Use `WebviewWindow` from `@tauri-apps/api/webviewWindow` to open settings
- Settings window: ~600x500, not resizable, titled "и®ҫзҪ®"
- Add a route guard or check: if opened as settings window, show settings directly

### Step 10: Update Global Styles (`src/assets/styles.css`)
- Add sidebar transition styles
- Add tab bar styles
- Add toolbar styles
- Ensure dark theme consistency

## Key Design Decisions

1. **No UI framework** - Keep pure CSS, maintain lightweight
2. **Lucide icons** - Tree-shakable, consistent, modern
3. **Tab persistence** - Tabs live in Pinia store (session only, not persisted)
4. **Sidebar state** - Default collapsed, stored in localStorage
5. **Settings window** - Separate Tauri WebviewWindow, loads `/settings` route
